# Less Annoying CRM API Documentation - Relationships

## Overview
Relationships connect two contacts or companies together for tracking purposes like family connections or referrals. Each relationship includes an optional note explaining the connection type.

**Key Features:**
- Bi-directional linking between contacts
- Visible on both contact records
- Alternative to structured "contact link" custom fields

---

## API Functions

### 1. CreateRelationship
**Purpose:** Creates a new relationship between two contacts/companies

**Parameters:**
- `ContactId1` (Text): First contact/company ID
- `ContactId2` (Text): Second contact/company ID
- `Note` (Text): Description of relationship nature

**Response:**
```json
{
  "RelationshipId": "4073197535295515471468544157937"
}
```

---

### 2. EditRelationship
**Purpose:** Modifies an existing relationship note

**Parameters:**
- `RelationshipId` (Uid, Required): Relationship identifier
- `Note` (Text, Required): Updated relationship description

**Response:** No return value (success indicated by absence of error)

---

### 3. DeleteRelationship
**Purpose:** Removes a relationship

**Parameters:**
- `RelationshipId` (Uid, Required): Relationship identifier

**Response:** No return value (success indicated by absence of error)

---

### 4. GetRelationship
**Purpose:** Retrieves single relationship details

**Parameters:**
- `RelationshipId` (Uid, Required): Relationship identifier

**Response:**
```json
{
  "RelationshipId": "...",
  "ContactId1": "...",
  "ContactId2": "...",
  "Note": "...",
  "CreatedBy": "...",
  "DateCreated": "...",
  "DateEdited": "...",
  "Contact1MetaData": {"Name": "..."},
  "Contact2MetaData": {"Name": "..."},
  "CreatedByMetaData": {"FirstName": "...", "LastName": "..."}
}
```

---

### 5. GetRelationshipsAttachedToContact
**Purpose:** Returns all relationships for a specific contact

**Parameters:**
- `ContactId` (Uid, Required): Contact identifier
- `MaxNumberOfResults` (Number, Default: 500): Results per call (max 10,000)
- `Page` (Number, Default: 1): Pagination control

**Response:**
```json
{
  "HasMoreResults": boolean,
  "Results": [
    {relationship objects...}
  ]
}
```
