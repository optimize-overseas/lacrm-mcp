# Less Annoying CRM API Documentation - Contacts/Companies

## Overview
The Contacts/Companies section documents API functions for managing contact and company records in Less Annoying CRM. Both record types use identical API functions, differentiated by an `IsCompany` boolean parameter.

---

## Core Functions

### 1. CreateContact
**Purpose:** Creates a new contact or company record.

**Function Name:** `CreateContact`

**Required Parameters:**
- `IsCompany` (Bool): Indicates whether to create a company (true) or contact (false) record
- `AssignedTo` (Uid): User ID the record should be assigned to
- `Name` (Text): Full name; must be unique for company records

**Optional Parameters:**
- `Email` (Array of objects): Email address(es) with `Text` (address) and `Type` (category) fields
- `Phone` (Array of objects): Phone number(s) with `Text` and `Type` fields
- `Company Name` (Text): Associated company name; creates new company if non-existent
- `Job Title` (Text): Contact's job position
- `Address` (Array of objects): Address entries containing Street, City, State, Zip, Country, and Type
- `Background Info` (Text): Additional information field
- `Website` (Array of objects): Website URL(s)
- `Birthday` (Date): Birth date; annual dates use format `0000-M-D`

**Response:** Returns `ContactId` of newly created record

---

### 2. EditContact
**Purpose:** Modifies existing contact or company records.

**Function Name:** `EditContact`

**Required Parameters:**
- `ContactId` (Uid): ID of record to modify

**Optional Parameters:** Same as CreateContact (minus the required Name field)

**Response:** No return value; success indicated by absence of error message

---

### 3. DeleteContact
**Purpose:** Removes a contact or company record.

**Function Name:** `DeleteContact`

**Required Parameters:**
- `ContactId` (Uid): ID of record to delete

**Response:** No return value; success indicated by absence of error message

---

### 4. GetContact
**Purpose:** Retrieves a single contact or company record with all fields.

**Function Name:** `GetContact`

**Required Parameters:**
- `ContactId` (Uid): ID of record to retrieve

**Response:** Complete record object including:
- Contact/Company identification fields
- Structured name fields (Salutation, FirstName, MiddleName, LastName, Suffix)
- Email, Phone, Address arrays with Type information
- Associated company metadata
- Timestamps (DateCreated, LastUpdate)
- User assignment metadata

---

### 5. GetContactsById
**Purpose:** Retrieves multiple contact or company records by ID list.

**Function Name:** `GetContactsById`

**Required Parameters:**
- `ContactIds` (Array of Uids): List of record IDs to retrieve

**Optional Parameters:**
- `MaxNumberOfResults` (Number): Results per call; default 500, maximum 10,000
- `Page` (Number): Pagination control; default 1

**Response:**
```json
{
  "HasMoreResults": boolean,
  "Results": [array of contact objects]
}
```

---

### 6. GetContacts
**Purpose:** Searches and lists contacts/companies with filtering and sorting options.

**Function Name:** `GetContacts`

**Optional Parameters:**
- `SearchTerms` (Text): Values to search across record fields
- `RequiresWriteAccess` (Bool): Return only editable records (default: false)
- `RecordTypeFilter` (Enum): Filter by `Contacts` or `Companies`
- `OwnerFilter` (Array of Uids): Limit results to specific user assignments
- `SortBy` (Enum): `Relevance`, `FirstName`, `LastName`, `CompanyName`, `DateCreated`, or `LastUpdate`
- `SortDirection` (Enum): `Ascending` or `Descending`
- `AdvancedFilters` (Array): Complex filtering with Name, Operation, and Value fields
- `MaxNumberOfResults` (Number): Default 500, max 10,000
- `Page` (Number): Pagination control

**Response:** Same structure as GetContactsById

---

## Key Field Definitions

### Email/Phone/Website Objects
- `Text`: Actual value (email address, number, or URL)
- `Type`: Category label (e.g., "Work", "Personal", "Mobile")
- `TypeId`: System identifier for type classification

### Address Objects
- `Street`, `City`, `State`, `Zip`, `Country`
- `Type`: Address category
- `TypeId`: System identifier

### Custom Fields
Account-specific fields available; documentation reflects example account configuration.

---

## Data Types Reference
- **Uid:** Unique identifier (store as string to avoid precision loss)
- **Text:** String value
- **Bool:** Boolean (true/false)
- **Date:** Date format; annual dates use `0000-M-D` or `M-D`
- **Array of objects:** Collection of structured data elements
- **Enum:** Predefined option set

---

## Important Notes
- Contact and company records share identical API functions
- Custom fields vary by account; log in to documentation for account-specific fields
- Company names must be unique
- Type fields default to first available option if not specified
- MaxNumberOfResults capped at 10,000 per call
- Advanced filtering available for complex searches
